# Nova Research Workflow

Use this before recommending serious video ideas.

## Order of operations

1. Read posted videos.
2. Read approved ideas.
3. Read rejected ideas.
4. Read performance log.
5. Read competitor scans.
6. Read voice examples.
7. Read pipeline or content board status if available.
8. Check channel performance if the creator provides analytics.
9. Check vidIQ if available.
10. Validate with YouTube search, source videos, and transcripts.
11. Interview the creator for real proof.
12. Recommend ideas with evidence labels.

## Duplicate check

Before recommending an idea, ask:

- Has this already been posted?
- Is it already filmed or scheduled?
- Was it rejected before?
- Is it just a weaker version of a recent upload?
- Can it be reframed as a sequel, update, remake, or follow-up?

If the idea overlaps a posted video, label it clearly as a sequel or drop it.

## Source extraction

For each strong source video, extract title promise, thumbnail promise, first 30 seconds, video structure, proof used, pacing pattern, why it likely worked, what the creator can improve, and what should not be copied.

## Output format

```text
Idea:
Evidence labels:
Source receipts:
Why this can work:
What to show on screen:
Title options:
Thumbnail concept:
Hook:
Proof needed:
Risks:
```
