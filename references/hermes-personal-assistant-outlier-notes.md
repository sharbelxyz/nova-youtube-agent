# Hermes personal assistant outlier notes

Use when researching or scripting Hermes Agent videos around personal assistants, 24/7 assistants, hacks, hidden features, or power-user workflows.

## Videos sampled in May 2026

- Nate Herk, `Hermes Agent: Zero to Personal AI Assistant (1 Hour Course)`, about 179K views.
- Metics Media, `Hermes Agent: The Ultimate Beginner’s Guide`, about 80K views.
- Alex Finn, `6 Hermes Agent use cases I promise will change your life`, about 56K views.
- Nate Herk n8n adjacent reference, `How to Build a Personal Assistant AI Agent in n8n`, about 155K views.

## What “personal assistant” means in this lane

The phrase is not mainly about a generic Jarvis demo. In the Hermes lane, it usually means:

- Hermes runs 24/7.
- Telegram is the visible command surface.
- The agent remembers preferences and work history.
- It creates or uses skills for repeated workflows.
- It runs scheduled jobs or crons that message the user first.
- It can use tools and manage real workflows.

Telegram is a major proof surface, not a minor integration. In sampled transcripts, Telegram appeared roughly 28 times in Nate Herk’s Hermes course, 28 times in the Metics Hermes guide, and 34 times in the n8n personal assistant video.

## Common content in high-view Hermes assistant videos

- Deploy Hermes on a VPS or private infrastructure.
- Connect an AI model provider.
- Set up Telegram.
- Explain skills, memory, crons, costs, and basic tool use.
- Position Hermes as different from a browser chatbot because it can run continuously and message first.
- Compare Hermes to Claude Code and OpenClaw when relevant.

This means generic beginner setup is crowded. For the creator, avoid making another plain beginner guide unless there is a new setup route or major product change.

## Strong Alex Finn use-case pattern

Alex Finn’s use-case video is closer to a power-user list than a beginner assistant video. The key ideas were:

1. Slash goal for long-running tasks, unlocked by meta-prompting first.
2. Kanban board as an agent task manager, start the day by giving Hermes work it can do while the human does human work.
3. Technical research and competitor breakdowns, Hermes controls a browser and inspects competing apps.
4. Memory wiki or work diary, a searchable log of memories, daily logs, subjects, and previous agent work.
5. General computer administrator, often framed with Tailscale and multi-device access.
6. Morning priority prompt, Hermes asks every morning what matters most, suggests tasks it can handle, and updates memory.

## the creator adaptation

Do not copy the builder-heavy examples one for one. Adapt the pattern to the creator’s operator, YouTube, and business workflows:

- Morning priority prompt for content and operator priorities.
- Work diary or memory wiki for YouTube decisions, strategy changes, and reusable lessons.
- Telegram as the command center.
- Scheduled jobs for AI news, X reaction opportunities, YouTube trend scans, and competitor outliers.
- Subagents for parallel research, for example vidIQ, YouTube validation, competitor scan, and channel history.
- Skills for repeatable workflows, especially Nova.
- Decision filter before filming, using vidIQ, YouTube outliers, and recent channel performance.
- Workspace updater, for example Notion Content Board, scripts, research docs, and plans.
- Computer administrator, grounded in files, logs, local projects, and setup verification.
- Safety boundaries for external actions, public messages, trading, spending, credentials, and account details.

## Transcript-analysis workflow lesson

When an outlier claim says a broad category is working, pull transcripts and inspect what the videos actually deliver before scripting. Count repeated terms and extract actual use cases, not just titles. Separate:

- Packaging promise.
- Actual demos shown.
- Repeated feature claims.
- Setup content versus power-user workflows.
- What is crowded versus what the creator can uniquely prove.
