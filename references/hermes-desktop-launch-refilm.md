# Hermes Desktop launch refilm pattern, 2026-06-02

Use when Sharbel has already filmed or drafted a Hermes Agent video and Nous ships an official product update that changes the packaging or demo path.

## Durable lesson

When an official Hermes announcement invalidates a filmed script, do not make a small intro patch. Treat it as a refilm and rebuild the script around the new product reality.

For Hermes Desktop launch coverage, the winning frame is:

- Hermes Desktop is not just a prettier chat app.
- It is a native command center for the same Hermes Agent core.
- Same config, same API keys, same sessions, same skills, same memory.
- Desktop makes the existing power-user workflows legible: files, tool output, settings, skills, cron, profiles, messaging, agents, command center.
- The desktop app complements CLI, Telegram, cron, and webhooks. It does not replace them.

## Verified install paths from docs

Existing Hermes install:

```bash
hermes desktop
```

Install CLI plus desktop:

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash -s -- --include-desktop
```

Project-specific launch:

```bash
hermes desktop --cwd <path>
```

## Script positioning to reuse

- Open with the official Nous tweet on screen, then cut to docs or download page.
- Explain that Hermes had power-user surfaces first: CLI, Telegram, cron, webhooks, skills, profiles, tools.
- Desktop changes the entry point because it makes those surfaces visible and easier to manage.
- Strong line: "Hermes was already powerful. Hermes Desktop makes it legible."
- Strong line: "Not a prettier chat app. A native control room for an agent that already had the power underneath."

## Demo beats

1. Official tweet.
2. Desktop docs/download page.
3. Install command or `hermes desktop` for existing users.
4. Chat with live tool output.
5. Drag-and-drop files and right preview rail.
6. File browser with real project folder.
7. Settings: providers, models, tools, credentials, MCP, gateway, sessions.
8. Skills as reusable SOPs.
9. Cron and triggers as background assistant layer.
10. Profiles for specialist agents.
11. Messaging gateway as mobile/omnichannel interface.

## Style constraints

- No em dashes.
- Do not use "most people use/think" framing.
- Avoid abstract "operating system" hype unless grounded in visible UI and workflows.
- For a refilm request, deliver a complete filming script plus SEO package, not only a change list.
