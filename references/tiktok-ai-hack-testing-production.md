# TikTok AI Hack Testing Production Pattern

Use when Sharbel wants to make a YouTube video around viral TikTok AI hacks/tools, especially a format like `I Tested the Most Viral TikTok AI Hacks`.

## Core format

Do not frame it as a generic tools list. Frame it as TikTok promises being tested:

- TikTok claim: what the viral clip promises
- Real test: the operator task Sharbel will run
- Result: screen-recorded output
- Verdict: Fake, Mid, Real, or Keeper

This maps better to proven YouTube formats like `I tested the most viral TikTok hacks/products` because each segment has built-in tension.

## Verdict system

Use a graded verdict, not simple pass/fail:

- Fake: misleading, broken, or only works in a cherry-picked demo
- Mid: works technically, but not useful enough
- Real: creates something usable with normal cleanup
- Keeper: Sharbel would actually use it again

Score each hack 1 to 10 across:

1. Promise match: did it do what TikTok claimed?
2. Output quality: is the result usable, not just impressive?
3. Speed: did it save meaningful time?
4. Workflow fit: would Sharbel use it again after filming?

Mapping:

- 1 to 3: Fake
- 4 to 5: Mid
- 6 to 8: Real
- 9 to 10: Keeper

## Important material-prep rule

When preparing tests, do not give placeholders like:

- `[Paste the source pack above]`
- `[Paste the testing criteria]`
- `[Add the 10 links here]`

Sharbel expects copy-paste-ready prompts. Every prompt for Gamma, NotebookLM, Claude Code, Lovable, Manus, Google Opal, etc. must be self-contained and fully expanded with the source data, TikTok links, metrics, claims, test plan, and criteria. He should be able to copy one block into the tool with no assembly.

## What to prepare for each hack

For every hack/tool, produce a ready-to-paste block containing:

- TikTok URL
- creator handle
- views, likes, shares when verified
- TikTok claim
- on-camera test
- pass condition
- exact tool prompt
- screen recordings needed
- expected failure point
- verdict criteria

If a tool needs shared source data, duplicate the full source pack inside that tool's prompt instead of referencing another section.

## Good segment mix

For Sharbel, the strongest lineup mixes viral proof with operator relevance:

- one or two mega-viral practical creator/business tools
- one image/thumbnail hack
- one voice/audio hack
- one learning/research hack
- one or two Claude Code/operator hacks
- one app builder comparison
- one autonomous agent test
- one video generation visual segment

Avoid filling the list with generic broad AI-list TikToks. Broad list TikToks can prove category demand in the intro, but main segments should be specific tools or hacks that can be tested on camera.

## Script structure notes

Use the scriptwriting PDF framework invisibly:

- confirm the click immediately
- start each segment with the TikTok promise
- state skepticism in one line
- run a real test
- give a sharp verdict
- re-hook into the next test

From the hook onward, deliver clean spoken prose only. No internal framework labels in the final script.
